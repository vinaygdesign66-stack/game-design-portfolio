# UI / HUD

## Overview

The user interface of **SAVIOUR** is designed to be clean, cinematic, and immersive. The HUD provides only essential information during gameplay, allowing players to focus on exploration, combat, and storytelling.

---

# Main Menu

The Main Menu is the first screen players see when launching the game.

## Options

* New Game
* Continue
* Load Game
* Settings
* Credits
* Exit

---

# Pause Menu

Players can pause the game at any time outside of cutscenes.

## Options

* Resume
* Restart Checkpoint
* Inventory
* Skills
* Weapon Upgrades
* Objectives
* Settings
* Save Game
* Quit to Main Menu

---

# Heads-Up Display (HUD)

The HUD displays important gameplay information without overcrowding the screen.

## HUD Elements

* Health Bar
* Stamina Bar
* Bracelet Energy
* Weapon Icon
* Arrow Count
* Mini Objective
* Compass
* Interaction Prompt

---

# Health Bar

Displays the Saviour's current health.

## Features

* Green = Healthy
* Yellow = Low Health
* Red = Critical Health

Health can be restored using healing items or by resting at sacred shrines.

---

# Stamina Bar

Used for actions such as:

* Sprinting
* Dodging
* Climbing
* Charged Attacks

The stamina bar automatically regenerates when not in use.

---

# Bracelet Energy

Displays the remaining divine energy stored in the Blue Bracelet.

Used for:

* Divine Abilities
* Special Attacks
* Bracelet Powers
* Enhanced Combat Skills

Energy is restored by defeating enemies, collecting relics, or interacting with sacred shrines.

---

# Weapon Display

Shows the currently equipped weapon.

Weapons include:

* Divine Sword
* Guardian Spear
* Sacred Bow
* Golden Knife

Also displays:

* Current weapon level
* Special ability availability

---

# Ammo Counter

Only appears when using the Sacred Bow.

Displays:

* Current Arrow Count
* Maximum Arrows
* Selected Arrow Type

Arrow Types

* Standard Arrow
* Fire Arrow
* Ice Arrow
* Explosive Arrow
* Divine Arrow

---

# Crosshair

The crosshair appears only while aiming the Sacred Bow or interacting with ranged mechanics.

It changes color when aiming at:

* Enemies
* Weak Points
* Interactive Objects

---

# Objective Tracker

Located in the upper-left corner.

Displays:

* Current Main Objective
* Optional Objectives
* Puzzle Progress
* Boss Objectives

Example:

* Rescue the Child
* Defeat the Crane Commander
* Find the Emerald Fang

---

# Compass

Located at the top of the screen.

Shows:

* Cardinal Directions
* Objective Marker
* Important Locations
* Quest Indicators

---

---

# UI Design Goals

The UI is designed to:

* Keep the screen clean and cinematic.
* Display only essential gameplay information.
* Minimize distractions during exploration and combat.
* Support both keyboard/mouse and controller players.
* Maintain a mythological visual style inspired by ancient symbols and divine artifacts.
